## 1. Virtual DOM(Document Object Model)이란?

> Virtual DOM (VDOM)은 UI의 이상적인 또는 "가상"적인 표현을 메모리에 저장하고 ReactDOM과 같은 라이브러리에 의해 "실제" DOM과 동기화하는 프로그래밍 개념
>
> — React 공식 문서

- 실제 DOM과 같은 내용을 담고 있는 복사본이다.
- 실제 DOM의 복사본이기 때문에, 실제 DOM의 모든 element과 속성을 공유한다.
- 복사본은 실제 DOM이 아닌 JS 객체 형태로 메모리 안에 저장되어 있다.

## 2. 사용하는 이유

만약 폰트의 컬러만 바꾸고 싶어도, 다음과 같이 DOM 조작을 필요로 하게 된다.

```js
document.querySelector('#title').style.color = "red";
```

브라우저는 HTML을 탐색해 해당 Element를 찾고, 해당 Element와 자식 Element들을 DOM에서 제거한다.

이후 새롭게 수정된 Element로 이를 교체.

CSS는 이 과정 이후 다시 계산하여 결과적으로는 레이아웃 정보를 알맞게 수정한다.

새롭게 계산된 내용이 브라우저에 그려지는 방식으로 플로우가 진행되게 된다.

가상돔은 메모리에 저장되어 있는 자바스크립트 객체에 불과하기 때문에(실제 브라우저를 접근하지 않는다.) 가상돔에 접근하고 수정하는 것은 매우 가볍고 빠른 작업이 되는 것이다.
